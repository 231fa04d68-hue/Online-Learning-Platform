Introduction to MongoDB

MongoDB is a popular NoSQL database that stores data in flexible, JSON-like documents. It allows for dynamic schemas, making it ideal for modern web applications. Here's an extensive guide to understand MongoDB basics.

What is MongoDB?

MongoDB is a document-oriented database that provides high performance, high availability, and easy scalability. Unlike traditional relational databases, it does not use tables and rows but stores data in collections of BSON documents.

Key Features

  • Document-based storage
  • Dynamic schemas
  • High scalability
  • Rich query language
  • Replication and sharding support

MongoDB Data Model

Data is stored in BSON documents, which are binary-encoded JSON documents. Documents consist of field and value pairs.

Example Document

{
  "name": "John Doe",
  "age": 30,
  "email": "john@example.com",
  "skills": ["JavaScript", "MongoDB", "Node.js"],
  "address": {
    "street": "123 Elm St",
    "city": "Somewhere",
    "zip": "12345"
  }
}

Installing MongoDB

You can install MongoDB Community Server on your machine from the official website. Alternatively, use cloud services like MongoDB Atlas.

Basic Commands

MongoDB Queries

This course dives deeper into querying MongoDB databases including CRUD operations, filtering, projections, and aggregation pipelines.

Create: Insert Documents

db.users.insertOne({ name: "Bob", age: 32, status: "active" });
db.users.insertMany([
  { name: "Carol", age: 28 },
  { name: "Dave", age: 45 }
]);

Read: Find Documents

Find documents using queries with comparison operators.

db.users.find({ age: { $gt: 30 } });
db.users.find({ status: "active" });
db.users.find({}, { name: 1, age: 1 }); // projection to include only name and age

Update Documents

db.users.updateOne({ name: "Bob" }, { $set: { status: "inactive" } });
db.users.updateMany({ age: { $lt: 30 } }, { $inc: { age: 1 } });

Delete Documents

db.users.deleteOne({ name: "Dave" });
db.users.deleteMany({ age: { $gte: 40 } });

Aggregation Framework

Powerful tool for data aggregation and transformation.

db.orders.aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: "$customerId", totalAmount: { $sum: "$amount" } } },
  { $sort: { totalAmount: -1 } }
]);

Indexes

Improve query performance by creating indexes.

db.users.createIndex({ name: 1 });
db.users.getIndexes();

Conclusion

Mastering MongoDB queries allows you to efficiently manipulate and retrieve data according to your application's needs.

Express.js - Setting Up Express

Express.js is a minimalist web framework for Node.js, used to build web applications and APIs efficiently.

Installing Express

npm init -y
npm install express

Creating a Basic Server

const express = require('express');
const app = express();
const PORT = 3000;

// Middleware to parse JSON bodies
app.use(express.json());

app.get('/', (req, res) => {
  res.send('Welcome to Express.js Server!');
});

app.listen(PORT, () => {
  console.log(Server running on http://localhost:${PORT});
});

Understanding Middleware

Middleware are functions that execute during the request-response cycle. They can modify requests, responses, or terminate the cycle.

// Example middleware
app.use((req, res, next) => {
  console.log(${req.method} request for ${req.url});
  next();
});

Routing Basics

app.get('/about', (req, res) => {
  res.send('About Page');
});

app.post('/data', (req, res) => {
  console.log(req.body);
  res.json({ message: 'Data received' });
});

Serving Static Files

Use express.static to serve images, CSS, and JavaScript files.

app.use(express.static('public'));

Error Handling Middleware

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});

Organizing Routes

Use express.Router() to modularize routes.

const router = express.Router();

router.get('/users', (req, res) => {
  res.send('Users List');
});

app.use('/api', router);

Express.js - RESTful APIs

Learn to create RESTful APIs using Express by handling routes, HTTP methods, controllers, and middleware.

REST API Basics

REST stands for Representational State Transfer. We use HTTP methods like GET, POST, PUT, DELETE to perform CRUD operations.

Example: Simple Users API

const express = require('express');
const app = express();
app.use(express.json());

let users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' }
];

// Get all users
app.get('/users', (req, res) => {
  res.json(users);
});

// Get user by ID
app.get('/users/:id', (req, res) => {
  const user = users.find(u => u.id === parseInt(req.params.id));
  if (!user) return res.status(404).send('User not found');
  res.json(user);
});

// Create new user
app.post('/users', (req, res) => {
  const user = {
    id: users.length + 1,
    name: req.body.name
  };
  users.push(user);
  res.status(201).json(user);
});

// Update user
app.put('/users/:id', (req, res) => {
  const user = users.find(u => u.id === parseInt(req.params.id));
  if (!user) return res.status(404).send('User not found');
  user.name = req.body.name;
  res.json(user);
});

// Delete user
app.delete('/users/:id', (req, res) => {
  users = users.filter(u => u.id !== parseInt(req.params.id));
  res.status(204).send();
});

const PORT = 3000;
app.listen(PORT, () => console.log(API running on port ${PORT}));

Middleware for Validation

function validateUser(req, res, next) {
  if (!req.body.name || req.body.name.length < 3) {
    return res.status(400).send('Name is required and should be at least 3 characters.');
  }
  next();
}

app.post('/users', validateUser, (req, res) => {
  // Create user logic here
});

Using Express Router

const router = express.Router();

router.route('/users')
  .get((req, res) => { /* ... */ })
  .post(validateUser, (req, res) => { /* ... */ });

app.use('/api', router);

Response Status Codes

  • 200 OK - Successful GET or PUT
  • 201 Created - Successful POST creating a resource
  • 204 No Content - Successful DELETE with no response body
  • 400 Bad Request - Validation error
  • 404 Not Found - Resource does not exist

Best Practices

  • Separate route logic and controllers
  • Use middleware for validation and authentication
  • Use proper HTTP status codes
  • Keep APIs stateless

React.js - JSX & Components

React is a popular frontend library for building user interfaces using components and JSX syntax.

What is JSX?

JSX is a syntax extension that looks like HTML but is used inside JavaScript to describe UI elements.

const element = <h1>Hello, world!</h1>;

Creating Functional Components

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

export default Greeting;

Using Components

import React from 'react';
import Greeting from './Greeting';

function App() {
  return (
    <div>
      <Greeting name="Alice" />
      <Greeting name="Bob" />
    </div>
  );
}

export default App;

Class Components (Legacy)

import React, { Component } from 'react';

class Welcome extends Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

export default Welcome;

JSX Rules

  • Must return a single parent element
  • Use camelCase for attributes (e.g. className, onClick)
  • JavaScript expressions wrapped in curly braces {}
  • Self-close tags: <img />, <br />

Rendering Elements

import React from 'react';
import ReactDOM from 'react-dom/client';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);

React.js - State & Props

State and props allow React components to manage dynamic data and communication between components.

Props

Props are read-only inputs passed from parent to child components.

function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

State

State is mutable data managed inside the component.

Using useState Hook (Functional Components)

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

Class Component State

class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}

Component Communication

Parent passes data via props; child can notify parent using callback props.

function Parent() {
  const [message, setMessage] = useState('');

  const handleChildData = (data) => {
    setMessage(data);
  };

  return (
    <div>
      <Child onSend={handleChildData} />
      <p>Message: {message}</p>
    </div>
  );
}

function Child({ onSend }) {
  return <button onClick={() => onSend('Hello Parent!')}>Send Message</button>;
}

Best Practices

  • Keep state minimal and local when possible
  • Lift state up to common ancestor when needed
  • Use prop-types or TypeScript for props validation

Node.js - Working with Files & Modules

Node.js provides modules for interacting with the file system and organizing your code.

The fs Module

Use fs to read and write files asynchronously or synchronously.

const fs = require('fs');

// Read file asynchronously
fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Write file asynchronously
fs.writeFile('output.txt', 'Hello Node.js', err => {
  if (err) throw err;
  console.log('File saved!');
});

The path Module

Helps work with file and directory paths.

const path = require('path');

const fullPath = path.join(__dirname, 'files', 'example.txt');
console.log(fullPath);

Creating and Exporting Modules

// math.js
function add(a, b) {
  return a + b;
}

module.exports = { add };

// app.js
const math = require('./math');
console.log(math.add(2, 3)); // 5

Using ES6 Modules (experimental)

// package.json
{
  "type": "module"
}

// math.mjs
export function add(a, b) {
  return a + b;
}

// app.mjs
import { add } from './math.mjs';
console.log(add(2, 3));

Working with Streams

Streams allow processing large data piece by piece.

const fs = require('fs');

const readStream = fs.createReadStream('largefile.txt', 'utf8');
readStream.on('data', chunk => {
  console.log('New chunk:', chunk);
});
